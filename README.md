# LearningGitAndGithub
## A Test Repo for learning the concepts of git.

### Notes:

#### Basics:
* a distributed file version control system
* Stores the current version, along with the entire history of files within a repository/project

##### Local Workflow
1. Create/Change files
2. Collect one or more changes together in a 'commit'
3. Save that set of changes to the history of a dev line
4. Make new versions of the codebase available to others

##### Project Files
* When a file is added, its 'untracked' or the repo hasnt been told to track it yet.
* `git add filename.file` has the repo track the file
